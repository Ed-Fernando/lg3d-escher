- return the address of the specified pointer

C SPECIFICATION

PARAMETERS

       _param1  Specifies  the  array  or  buffer  pointer  to be
                returned.            Symbolic           constants
                GL_COLOR_ARRAY_POINTER,
                GL_EDGE_FLAG_ARRAY_POINTER,
                GL_FEEDBACK_BUFFER_POINTER,
                GL_INDEX_ARRAY_POINTER,  GL_NORMAL_ARRAY_POINTER,
                GL_TEXTURE_COORD_ARRAY_POINTER,
                GL_SELECTION_BUFFER_POINTER,                  and
                GL_VERTEX_ARRAY_POINTER are accepted.

       _param2  Returns the pointer value specified by _param1.

DESCRIPTION

       glGetPointerv  returns  pointer  information. _param1 is a
       symbolic constant indicating the pointer to  be  returned,
       and  _param2  is a pointer to a location in which to place
       the returned data.

NOTES

       glGetPointerv is available only if the GL version  is  1.1
       or greater.

       The pointers are all client-side state.

       The initial value for each pointer is 0.

       When   the  GL_ARB_multitexture  extension  is  supported,
       querying the  GL_TEXTURE_COORD_ARRAY_POINTER  returns  the
       value for the active client texture unit.

ERRORS

       GL_INVALID_ENUM is generated if _param1 is not an accepted
       value.
